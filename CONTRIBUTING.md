# Contributing to Edge Detection Viewer

Thank you for your interest in contributing! This project is part of a technical assessment.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages
7. Push to your fork
8. Create a Pull Request

## Commit Message Guidelines

Follow conventional commits:
- `feat: Add new feature`
- `fix: Fix bug in camera processing`
- `docs: Update README`
- `refactor: Improve native code structure`
- `test: Add unit tests`
- `chore: Update dependencies`

## Code Style

### Java/Kotlin
- Follow Android code style guidelines
- Use meaningful variable names
- Add comments for complex logic

### C++
- Follow Google C++ Style Guide
- Use RAII principles
- Handle exceptions properly

### TypeScript
- Use strict mode
- Define interfaces for data structures
- Add JSDoc comments

## Testing

Before submitting PR:
1. Build succeeds without errors
2. App runs on physical device
3. Edge detection works correctly
4. Web viewer displays frames properly
5. No memory leaks detected

## Questions?

Open an issue for discussion before major changes.
